## 主要修复

### 1. 配音 Pipeline 回调签名错误修复

**问题：** `DubbingInterfaceThread._on_pipeline_progress` 回调函数签名与 `DubbingPipeline` 实际调用不匹配，导致 `TypeError: '>' not supported between instances of 'str' and 'int'`

**根本原因：**
- Pipeline 的回调规范：`callback(progress: int, message: str)` 两个参数
- Thread 中错误定义为：`(current: int, total: int, message: str)` 三个参数
- 当 pipeline 调用 `cb(2, "loading subtitles")` 时，字符串被当作 `total` 参数，触发类型错误

**修复内容：**
- 将 `_on_pipeline_progress(current, total, message)` 改为 `(progress, message)`
- 直接透传进度百分比，不再重新计算映射
- Pipeline 内部已管理完整的 0-100% 进度

**验证结果：** ✅ 配音进度正常显示（2% → 8% → 10%-85% → 88% → 100%）

**修改文件：** `videocaptioner/ui/thread/dubbing_interface_thread.py:181`

### 2. DubbingResult 属性错误修复

**问题：** `TypeError: DubbingInterfaceThread.finished[str].emit(): argument 1 has unexpected type 'DubbingResult'`

**根本原因：**
- UI 信号定义：`finished = pyqtSignal(str)` 只接受字符串
- Thread 代码尝试传递：`self.finished.emit(output)` 其中 `output` 是 `DubbingResult` 对象
- 属性名错误：代码使用了 `output.output_audio_path`，实际应为 `output.audio_path`

**修复内容：**
- 将 `self.finished.emit(output)` 改为 `self.finished.emit(str(output.audio_path))`
- 两处修复：字幕模式（line 104）和文案模式（line 164）

**验证结果：** ✅ 配音完成后 UI 正常显示成功消息

**修改文件：** `videocaptioner/ui/thread/dubbing_interface_thread.py:104, 164`

### 4. ElevenLabs API 404 错误修复

**问题：** ElevenLabs 配音报错 `status_code: 404`，请求被发送到 `api.openai.com` 而非 `api.elevenlabs.io`

**根本原因：**
- `cfg.dubbing_api_base` 默认值：`https://api.openai.com/v1`
- `TaskFactory` 对所有 provider 使用相同的 `base_url`
- ElevenLabs SDK 需要 `base_url=""` 或 `None` 才会使用内置的正确端点

**修复内容：**
```python
if provider == "openai":
    if not base_url:
        base_url = "https://api.openai.com/v1"
elif provider == "elevenlabs":
    model = "eleven_flash_v2_5"
    base_url = ""  # SDK 使用内置 api.elevenlabs.io
```

**验证结果：** ✅ ElevenLabs 请求不再发到错误的端点

**修改文件：** `videocaptioner/ui/task_factory.py:315-334`

## 新增功能

### 6. 语速控制滑块（完整版）

**需求：** 用户希望能够控制配音语速，且需要精确输入和快速重置

**实现内容：**

1. **配置项**（`config.py`）：
   ```python
   dubbing_speed = RangeConfigItem("Dubbing", "Speed", 1.0, RangeValidator(0.5, 2.0))
   ```

2. **UI 控件**（`dubbing_interface.py`）：
   - **Slider 滑块**：范围 50-200（0.5x-2.0x），宽度 180px
   - **DoubleSpinBox 输入框**：范围 0.5-2.0，步进 0.05，显示 "x" 后缀，宽度 80px
   - **ToolButton 重置按钮**：图标 FIF.SYNC，一键恢复 1.0x

3. **交互逻辑**：
   - 滑块 ↔ 输入框双向同步（使用 `blockSignals()` 防止循环触发）
   - `_on_speed_slider_changed(value)`: 滑块 → 输入框
   - `_on_speed_spin_changed(value)`: 输入框 → 滑块
   - `_on_speed_reset()`: 重置按钮 → 1.0x

4. **配置持久化**：
   - `load_config()`: 加载 `cfg.dubbing_speed.value` 到输入框（滑块自动同步）
   - `save_config()`: 保存 `speed_spin.value()` 到配置

5. **Pipeline 集成**：
   - `TaskFactory.create_dubbing_config()` 读取语速配置传递给 `DubbingConfig.speed`

**UI 布局：**
```
语速: [━━━●━━━━] [1.00x ▼] [🔄]
      滑块        输入框   重置
```

**用户操作方式：**
1. 拖动滑块 → 输入框实时更新
2. 输入框输入精确值（如 1.25）→ 滑块自动移动
3. 点击重置 🔄 → 恢复 1.0x

## 技术要点

### 双向同步模式（滑块 ↔ 输入框）

```python
def _on_speed_slider_changed(self, value):
    speed = value / 100.0
    self.speed_spin.blockSignals(True)  # 防止循环
    self.speed_spin.setValue(speed)
    self.speed_spin.blockSignals(False)

def _on_speed_spin_changed(self, value):
    slider_value = int(value * 100)
    self.speed_slider.blockSignals(True)  # 防止循环
    self.speed_slider.setValue(slider_value)
    self.speed_slider.blockSignals(False)
```

### Provider 特定的 base_url 处理

不同 TTS provider 需要不同的 API 端点策略：
- **OpenAI**: 使用配置的 base_url 或默认值
- **ElevenLabs**: 强制留空让 SDK 使用内置端点
- **Edge TTS**: 无需 base_url（本地实现）
